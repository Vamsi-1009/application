Everything before this stage is organized as a reference — pick it up at any point, read the stage you need. This one is different: it's a linear, checkable path through the whole manual, plus a concrete project spec that touches all 19 stages, so finishing this ends in something you actually shipped.

The build-order checklist#

Each item is a real, testable outcome — not "read Stage X." Checking a box here saves to your browser only (same mechanism as the "mark stage complete" buttons); it's a personal build log, not a synced account.

Capstone project: TinyLink#

A deliberately small project that still touches every stage above — an authenticated link shortener. Small enough to actually finish, real enough that every decision in this manual matters.

Auth & accounts

Sign up with email OTP verification. Argon2id password hashing. Short-lived JWT + rotating refresh token.

Core feature

Create a short link tied to your account. Redirect endpoint is rate-limited and logs a click count.

Data layer

Postgres schema with a unique, indexed short-code column, migrations tracked in git.

Storage & CDN

Optional per-link custom preview image, uploaded via a presigned URL, served through a CDN.

Deployment

Two app instances behind a load balancer on a hardened VM, a real domain with HTTPS.

Operations

GitHub Actions auto-deploy, redacted structured logs, one alert wired up, one tested backup restore.

Definition of done

Ship it publicly reachable over HTTPS, then run the Stage 17-style AI security audit against your own code before calling it finished. If it finds something real, fix it — that last step is the difference between "it works" and "it's actually ready."
